> v0.0.5 是一项 Windows 稳定性 hotfix，重点修复发布版启动后队列无法载入，并完善悬浮窗、快捷取号、实时分享图和处理记录管理。

## Highlights

- **启动恢复正常**：数据库状态会在窗口与 IPC 命令创建前完成注册，修复启动后一直显示“正在整理”或提示 `state not managed` 的问题。
- **悬浮窗默认可见**：软件启动时自动显示悬浮窗；关闭主窗口后继续在后台运行并保留悬浮窗。
- **快捷取号窗口**：托盘菜单和 `Ctrl + Alt + N` 可直接打开独立新增取号页面，无需先展开完整主界面。
- **实时分享图**：每次复制都会读取最新队列快照，前方事项数与待处理总数会随队列实时更新。
- **加急自动前移**：事项首次标记加急时自动前移一个人工顺位，仍可继续使用上移、下移精细调整。

## 新增

- 用户手动添加的处理记录支持编辑和删除；系统自动生成的审计记录保持只读。
- 悬浮窗迷你模式固定提供“打开主界面”和“展开悬浮窗”两个独立入口。

## 改进

- 分享图右下角提示改为“当前有x个事项待处理，请耐心等待”。
- 缩小部门 / 团队、事项类型和对接人输入框与下拉选项面板之间的间距。
- 悬浮窗切换为迷你模式后显示放大图标，展开状态显示缩小图标。

## 修复

- 修复 `bootstrap` 调用时数据库状态尚未注册，导致队列无法载入的问题。
- 修复悬浮窗圆角外侧残留半透明矩形阴影的问题，圆角曲率调整为更平滑的 16px。
- 修复托盘“新增事项”必须先显示完整主窗口的问题。
- 修复分享图使用列表旧数据，队列变化后前方事项数未及时更新的问题。
- 修复开发模式引用不存在的 npm 脚本而无法启动的问题。

> [!WARNING]
> 安装 v0.0.5 前请从托盘正常退出旧版。升级不会主动删除 `%APPDATA%\in-line` 中的用户数据库。安装包目前未使用商业代码签名证书，Windows SmartScreen 可能提示未知发布者。
