# Changelog

本项目的所有重要变更均记录于此。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [Unreleased]

## [0.0.7] - 2026-07-18

### Changed

- 悬浮窗迷你模式和展开模式共用同一套固定工具栏，左侧拖动点和右侧三个窗口按钮在切换前后保持相同大小与位置。
- 悬浮窗固定为 420px 宽度并关闭原生窗口缩放，避免 Windows 为无标题栏窗口绘制额外边框。
- 悬浮窗恢复 Windows 原生投影，使窗口与桌面背景保持清晰层次。
- 移除 `Ctrl + Alt + N` 全局新增取号快捷键、界面提示及对应插件权限；新增事项继续通过主界面、悬浮窗或托盘菜单打开。

### Fixed

- 左侧九点拖动柄、顶部品牌空白区域、列表剩余空白区域和底部空白区域现在均可拖动悬浮窗。
- 移除悬浮窗最外侧的 CSS 边框、阴影和 Windows 原生缩放细线。
- 修复迷你模式遗留的左右内边距和元素间距导致工具栏图标切换时仍会位移的问题。

## [0.0.6] - 2026-07-18

### Added

- 待办队列、历史归档和回收站新增 Excel 式列筛选，支持按部门 / 团队、对接人、事项类型、当前状态和截止时间筛选。
- 截止时间筛选支持指定自然日，以及上午、中午、下午和晚上四个粗粒度时段。

### Changed

- 悬浮窗迷你模式与展开模式保持相同宽度，并改为无圆角直角窗口。
- 分享图右下角改为显示当前事项前方仍有多少个事项待处理，不再显示整个队列总数。
- 事项类型与要求完成时间控件改为同宽对齐，选项面板紧贴输入框显示。

## [0.0.5] - 2026-07-18

### Added

- 新增独立的快捷取号窗口：托盘菜单和系统级快捷键可直接打开表单，无需先显示完整主界面。
- 用户手动添加的处理记录支持编辑和删除；系统自动生成的审计记录仍保持只读。

### Changed

- 软件启动时同时显示主窗口与悬浮窗；关闭主窗口后继续在后台运行，并保留悬浮窗。
- 悬浮窗迷你模式固定保留“打开主界面”按钮，尺寸切换按钮会按当前状态显示缩小或放大图标。
- 首次将事项标记为加急时，事项自动前移一个人工顺位。
- 复制分享图时实时读取数据库队列快照，右下角显示“当前有x个事项待处理，请耐心等待”。

### Fixed

- 修复发布版启动时数据库状态注册过晚，导致 `bootstrap` 无法载入队列的问题。
- 修复悬浮窗圆角外侧残留半透明矩形阴影的问题，并将圆角曲率统一调整为更平滑的 16px。
- 缩短可输入下拉框与选项面板之间的间距。
- 修复开发模式启动命令引用不存在 npm 脚本的问题。

## [0.0.4] - 2026-07-17

### Added

- 事项类型和经办对接人新增可输入组合下拉框，点击即可查看预设值及已保存的自定义内容。
- 经办对接人保存后自动记忆，并在表单下方显示三个最常用联系人快捷项。
- 侧边栏新增“关于”页面，展示作者六朝声、GitHub 开源地址和运行中的实际版本号。
- 部门 / 团队、事项类型和对接人统一使用可输入下拉框，并支持删除不再使用的自定义选项。
- 要求完成时间新增确认按钮，以及半小时后、1 小时后、上午、中午、下午和下班前快捷选项。
- 可恢复备份新增删除操作，历史备份会统一整理为 `InLine-backup-日期时间-类型.db`。

### Changed

- 主界面侧边栏改为辨识度更高的雾蓝色，并重新校准导航、选中态和统计卡片对比度。
- 取号图片改为更紧凑的 800×980 竖版：放大号码、增加前方待处理事项数，状态胶囊会随文字宽度自适应。
- 取号图片提示语改为“将按当前队列顺序处理”，不再显示冗余的登记顺序说明。
- 悬浮窗顶部和底部改用与侧边栏一致的浅蓝色，并保留深蓝文字与蜜蜡黄品牌点缀。

### Fixed

- 修复复制取号图片时将 PNG 压缩字节误传为 RGBA 数据而导致复制失败的问题。
- 修复截止时间保存为 UTC 后，编辑表单直接截取字符串而显示错误时区时间的问题。
- 修复透明悬浮窗四角仍露出浅蓝色直角底边的问题，窗口现仅保留圆角轮廓。
- 修复主窗口任务栏图标仅有透明占位的问题，窗口和托盘均显式使用应用图标。
- 事件详情改为选填，未填写时不会阻止取号。

## [0.0.3] - 2026-07-17

### Fixed

- 修复 Windows 发布构建误显示黑色控制台窗口的问题。
- 修复数据库初始化失败时主界面无限停留在“正在整理队列”的问题，现会显示明确错误并允许重试。
- 修复悬浮窗“新增取号”无法可靠打开新增表单的问题。
- 修复悬浮窗右键“查看、编辑、改状态、加急”仅打开主窗口、未定位对应事项的问题。
- 修复前端事件监听销毁竞态和队列移动失败没有提示的问题。

### Changed

- 主界面左下角显示实际应用版本，并补充 Electron v0.0.1 与 Tauri 新版并存时的升级清理说明。

## [0.0.2] - 2026-07-17

### Added

- 新增持久化人工队列顺位，支持相邻事项上移和下移。
- 新增 1200×800 横版取号卡片，主列表和悬浮窗单击事项即可复制。
- 新增结构化状态历史、加急历史、数据库版本迁移和迁移前自动备份。
- 新增备份列表、安全恢复、系统级 `Ctrl + Alt + N` 快捷键及单实例运行。
- 新增透明叫号铃品牌资源，以及复制、上移、下移三枚统一 SVG / PNG 图标。

### Changed

- 桌面架构由 Electron 迁移至 Tauri 2，保留 React、TypeScript 和 SQLite。
- 队列改为人工顺位优先；加急、优先级和逾期仅作醒目标识，不再自动打乱顺序。
- 行内操作统一为复制、上移、下移，其余操作集中到右键菜单。
- 统一 Windows 中文字体、字号、行高、色板和 Lucide 界面图标。
- 应用定位和默认文案改为通用取号与事项队列工具，不再限定法务场景。
- README 改为直接使用用户提供的原始 PNG 品牌图，不再展示重绘版本。
- 开源协议统一为 GNU GPL v3，并移除 Electron 运行时、旧源码和无效的“300 DPI”描述。

### Fixed

- 修复 125%–200% Windows 缩放下字体错位、中文基线偏移和内容裁切。
- 修复复制图片入口和实现重复的问题，保证一次操作只生成一次图片。
- 修复悬浮窗迷你模式的透明剩余区域遮挡桌面鼠标操作。
- 修复取号和每日备份使用 UTC 日期造成的本地日期边界错误。
- 修复应用窗口、托盘、安装包和快捷方式未使用统一透明图标的问题。

### Security

- 前端不再执行任意 SQL，仅调用带输入长度、枚举和路径校验的受控命令。
- Tauri capability 仅开放实际所需权限；应用默认无遥测、无外部网络访问。
- 备份恢复前执行 SQLite 完整性校验，并先备份当前数据库；日志不记录完整敏感文本。
