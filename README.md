# In Line｜排着呢

In Line 是一款通用的 Windows 本地取号与事项队列工具。它适合个人或内部支持岗位记录任务、展示排队顺序、发送取号卡片，适合业务量较大的职能部门使用。

当前代码版本：`v0.0.6`（2026-07-18）。完整变更见 [CHANGELOG.md](CHANGELOG.md)。

## v0.0.6 已实现

- 每日自动取号，跨天显示 A、B … AA 前缀
- 主窗口与桌面悬浮窗共用同一人工队列
- 单击事项即复制紧凑的 800×980 竖版取号卡片，突出号码并显示前方待处理事项数
- 取号卡片每次复制时都会读取最新队列，并显示当前事项前方仍有多少项待处理
- 行内仅保留复制、上移、下移三个操作
- 右键查看、编辑、改状态、加急、完成、归档或移入回收站
- 部门 / 团队、对接人、事项类型、状态、优先级和截止时间
- 本地 SQLite 数据库、结构化状态历史和加急历史
- 部门 / 团队、事项类型和经办对接人使用统一的可输入下拉框，可自动记忆或删除自定义内容
- 表单下方显示三个最常用对接人快捷项
- 截止时间支持确认后保存，以及半小时后、1 小时后、上午、中午、下午和下班前快捷选项
- 事件详情为选填项
- “关于”页面展示作者六朝声、GitHub 开源地址和当前软件版本
- 迁移前备份、手动备份、统一备份命名、备份删除和安全恢复
- 真正的系统级 Ctrl + Alt + N 快捷键
- 托盘菜单和全局快捷键可直接打开独立新增取号窗口，无需先展开主界面
- 待办队列、历史归档和回收站均支持按部门、对接人、事项类型、状态和截止时间进行列筛选
- 截止时间支持按自然日，以及上午、中午、下午、晚上四个时段筛选
- 系统托盘、开机启动，以及宽度稳定的直角悬浮窗迷你模式
- 关闭主窗口后继续在后台运行并保留悬浮窗；处理记录支持编辑或删除手动条目
- 透明应用图标和统一 Lucide 界面图标
- 单实例、无遥测、无外部网络、受控 Tauri 命令

## 数据兼容与位置

v0.0.6 会继续使用：

`%APPDATA%\in-line\inline.db`

首次迁移旧数据库时，会先在 `backups` 目录生成 `InLine-backup-日期时间-before-migration.db`。旧格式备份会在启动时统一整理名称。应用更新或卸载不会主动删除用户数据库。

### 从旧版升级

1. 从托盘菜单正常退出旧版 In Line，确认任务管理器中不再有 `In Line.exe`。
2. 如果“设置 → 应用”仍显示 **In Line 0.0.1**，先卸载该 Electron 旧版。卸载不会删除 `%APPDATA%\in-line\inline.db`。
3. 删除或取消固定仍指向旧安装目录的快捷方式；旧版常见位置为 `D:\工具\In Line\In Line.exe`。
4. 安装并启动 v0.0.6；程序会先备份旧数据库，再在事务中执行迁移。
5. 在主界面左下角确认显示 `v0.0.6`，再核对原有事项、号码和日志。

迁移失败时会回滚数据库，不会按新项目清空原有数据。首次升级前仍建议保留一份手动备份。

如果主界面一直停在“正在整理队列”，修复后的程序会显示真实错误和“重新载入”按钮，不再无限等待；悬浮窗的“新增取号”会直接打开主界面的新增表单。

## 构建产物

Windows x64 安装包采用 NSIS 格式，使用系统 WebView2，不捆绑完整浏览器运行时。源码仓库不提交安装包二进制；正式安装包通过 GitHub Releases 提供。

v0.0.6 正式安装包：

- `In.Line_0.0.6_x64-setup.exe`
- 通过 GitHub Release 工作流构建并附带 SHA-256 校验值

## 开发

推荐环境：

- Windows 10 / 11 x64
- Node.js 20+
- Rust stable-msvc
- Visual Studio 2022 Build Tools（Desktop development with C++）
- Microsoft Edge WebView2 Runtime

```powershell
npm.cmd install
npm.cmd test
npm.cmd run build
npm.cmd run dist
```

前端为 React + TypeScript + Vite；桌面层为 Tauri 2 + Rust；数据层为 rusqlite + SQLite。

## 安全和隐私

- 纯本地应用，不会上传数据，不包含遥测或第三方统计。
- 前端不能执行任意 SQL，只能调用白名单命令。
- 日志不记录完整事项详情和内部备注。
- 备份恢复前会校验 SQLite 完整性，并自动备份当前数据。
